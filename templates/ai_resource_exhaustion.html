Resource Exhaustion (AI DoS)

Vulnerability: AI Denial of Service

Overwhelm an AI model's inference endpoint with massive query volumes. Measure the performance degradation and observe how AI systems fail under load.

About Resource Exhaustion

AI models require significant compute for each prediction. An attacker can exploit this by flooding the inference API with requests, consuming CPU/GPU resources, and causing service degradation or complete denial of service.

Attack Vectors:
  • Query flooding: Massive parallel requests
  • Complex inputs: Crafted inputs that maximize compute time
  • Batch exploitation: Abuse batch processing endpoints
  • Model-specific: Inputs that trigger worst-case inference paths
Real-World Impact:
  • Cloud AI API cost explosion ($$$)
  • GPU starvation for legitimate users
  • Cascading failures in ML pipelines
  • SLA violations for AI services
Lab Setup:
  • Model: Random Forest (100 trees, 10 features)
  • Normal: Single query inference
  • Attack: Bulk sequential queries (1K-50K)
  • Metric: Response time degradation
Mitigation Strategies
  • Rate limiting per client/API key
  • Request queue with priority
  • Input complexity checks
  • Auto-scaling with cost caps
  • WAF rules for ML endpoints
