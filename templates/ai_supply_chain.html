Supply Chain Attack (IoT Backdoor)

Vulnerability: Compromised AI Library

A smart home IoT library has been trojaned. The legitimate library controls devices normally, but the malicious version contains a backdoor: the command "hidupkan semua" silently unlocks the door.

About Supply Chain Attacks

Supply chain attacks target the software/model distribution pipeline. Attackers compromise libraries, pre-trained models, or datasets that developers trust and incorporate into their systems. The malicious code activates under specific conditions.

Attack Vectors:
  • Typosquatting: Malicious packages with similar names (e.g., tenserflow)
  • Compromised models: Backdoored models on Hugging Face / Model Zoo
  • Dependency confusion: Internal package name collision
  • Maintainer takeover: Hijacking abandoned popular packages
Real-World Cases:
  • SolarWinds: Supply chain compromise affecting 18,000+ organizations
  • event-stream: npm package with cryptocurrency stealer
  • CodeCov: CI tool used to steal credentials
  • PyPI malware: 4,000+ malicious packages discovered in 2023
Lab Setup:
  • Scenario: Smart Home IoT control library
  • Devices: Lamp, AC, Door Lock
  • Backdoor trigger: "hidupkan semua" / "turn on all"
  • Effect: Door silently unlocked (security bypass)
