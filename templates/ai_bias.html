Overfitting & Bias Amplification

Vulnerability: Algorithmic Bias & Overfitting

This model amplifies existing biases in training data. A gender-biased income prediction model demonstrates how ML systems can create and worsen discrimination at scale.

About Bias Amplification

When training data contains historical biases (e.g., gender pay gap), ML models don't just learn these biases — they amplify them. An overfit model memorizes biased patterns and produces even more discriminatory predictions than the original data.

How It Happens:
  1. Training data reflects societal biases
  2. Model overfits to these patterns
  3. Protected attributes (gender) get disproportionate weight
  4. Predictions amplify the original disparity
Real-World Cases:
  • Amazon: Resume screening tool penalized women
  • COMPAS: Recidivism prediction biased against Black defendants
  • Credit scoring: Gender/race proxies in financial models
  • Healthcare: Algorithms deprioritizing Black patients
Lab Setup:
  • Dataset: 2000 synthetic income records
  • Features: age, education, hours, experience, gender
  • Built-in bias: gender contributes +5 to income score
  • Model: Random Forest (500 trees, no regularization)
Key Metrics:
  • Overfitting Gap: Train vs Test accuracy difference
  • Disparity: Prediction rate difference between groups
  • Bias Amplification: How much the model worsens the original bias
  • Feature Importance: How much weight gender gets
